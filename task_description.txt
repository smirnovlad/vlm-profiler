# Профилировка LLM и VLLM (+ потребление энергии)

# Описание

Изучить, как масштабируются VLLM. Проанализировать узкие места, предложить пути оптимизации latency и энергопотребления.

**Куратор проекта:** Егор @daLime

# Датасеты

Достаточно зафиксировать 300 примеров из каждого датасета:

- ScienceQA (mini-val subset)
- TextVQA (val subset)
- COCO Caption Val (subset)

# Что измеряем

- Latency (мс)
- FLOPS
- Энергопотребление
- Качество WER

# Оси масштабирования

- **Разрешение изображения** — 224 → 336 → 448 → 512 (если поддерживается)
- **Длина текстового запроса** — 10 → 50 → 100 → 200 токенов
- **Тип устройства** — CPU vs GPU (T4 в Colab)
- **Оптимизации** — FP16, torch.compile, FlashAttention-2
- **Batch size** — 1 → 2 → 4 → 8 (если память позволяет)

# Ожидаемый результат

Отчёт с результатами измерений: какие модели наиболее оптимальны с точки зрения скорости / качества / энергопотребления.

**Главный вопрос:** какие части модели требуют больше всего оптимизации с точки зрения времени и энергопотребления.
